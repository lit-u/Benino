**CRITICAL placement:** This route MUST be placed AFTER `router.get('/', ...)` (line ~68) and BEFORE `router.get('/:slug', ...)` (line ~88). If placed after `/:slug`, Express will match "mine" as a slug parameter and query `WHERE slug = 'mine'` — returning 404 instead of the user's hotel list.

Implementation:
```javascript
// GET /mine — list hotels owned by authenticated user (MUST be before /:slug)
router.get('/mine', requireUser, async (req, res) => {
  try {
    const owner_id = req.authContext?.userId || req.auth?.identityId;
    if (!owner_id) return res.status(401).json({ success: false, error: 'Negalima nustatyti vartotojo tapatybes' });

    const { data, error } = await supabase
      .from('hotels')
      .select('id, slug, name, city, created_at, owner_email')
      .eq('owner_id', owner_id)
      .order('created_at', { ascending: false });

    if (error) return res.status(500).json({ success: false, error: error.message });
    return res.json({ success: true, hotels: data || [] });
  } catch (err) {
    console.error('GET /api/hotels/mine error:', err);
    return res.status(500).json({ success: false, error: err.message });
  }
});
```

Use the exact `owner_id` resolution expression (`req.authContext?.userId || req.auth?.identityId`) — same as existing `POST /` and `verifyOwner()`.
