**Change 2: Set min values after form creation**

After the line `const depInput = formEl.querySelector(...)` (around line 497), add this block:

```javascript
// Enforce min dates (CAL-04, CAL-05)
const todayISO = new Date().toISOString().split('T')[0];
if (arrInput) arrInput.min = todayISO;

function updateDepMin() {
  if (!arrInput || !depInput) return;
  const arrVal = arrInput.value;
  if (arrVal) {
    const nextDay = new Date(arrVal);
    nextDay.setDate(nextDay.getDate() + 1);
    depInput.min = nextDay.toISOString().split('T')[0];
    // If current departure is before new min, reset it
    if (depInput.value && depInput.value <= arrVal) {
      depInput.value = depInput.min;
    }
  }
}

// Set initial departure min
updateDepMin();

// Update departure min when arrival changes
if (arrInput) arrInput.addEventListener('change', updateDepMin);
```

**Change 3: Verify the existing change listener is preserved**

The existing line `[arrInput, depInput].forEach(el => el?.addEventListener('change', () => updatePriceSummary(formEl, room)));` must remain intact. The new `arrInput.addEventListener('change', updateDepMin)` is an ADDITIONAL listener — JS supports multiple listeners on the same event, they do not conflict.

Do NOT modify the POST body or validation logic — `arrival_date` and `departure_date` are already sent correctly (CAL-06 is already satisfied).
