-- Add owner_email to hotels
ALTER TABLE hotels ADD COLUMN IF NOT EXISTS owner_email text;

-- Create hotel_reservations table
CREATE TABLE hotel_reservations (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  hotel_id uuid NOT NULL REFERENCES hotels(id) ON DELETE CASCADE,
  room_id uuid REFERENCES hotel_rooms(id) ON DELETE SET NULL,
  guest_name text NOT NULL,
  guest_email text,
  guest_phone text,
  arrival_date date NOT NULL,
  departure_date date NOT NULL,
  nights integer NOT NULL CHECK (nights > 0),
  total_price numeric(10,2),
  message text,
  status text NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'confirmed', 'cancelled')),
  created_at timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX hotel_reservations_hotel_id_idx ON hotel_reservations(hotel_id);
CREATE INDEX hotel_reservations_status_idx ON hotel_reservations(status);

Add this method after `sendWelcomeEmail()` and before the closing `}` of the class:

```javascript
async sendReservationNotification({ ownerEmail, hotelName, roomName, guestName,
  guestEmail, guestPhone, arrivalDate, departureDate, nights, totalPrice, message }) {
  if (!this.resend) return { success: false, error: 'RESEND_API_KEY not configured' };
  try {
    const { data, error } = await this.resend.emails.send({
      from: this.from,
      to: ownerEmail,
      subject: `Nauja rezervacija - ${roomName}, ${hotelName}`,
      html: `<!-- reservation notification HTML -->`
    });
    if (error) throw error;
    return { success: true, messageId: data.id };
  } catch (err) {
    console.error('Failed to send reservation notification:', err);
    return { success: false, error: err.message };
  }
}
```

Key differences from sendVerificationEmail:
- Does NOT re-throw errors (returns `{ success: false }` instead) — caller uses fire-and-forget pattern
- Guard: `if (!this.resend) return { success: false, error: ... }` (same pattern as sendWelcomeEmail)
- HTML template uses amber/gold gradient header (`#f59e0b` to `#d97706`) with `.detail-row` layout — use the exact template from 06-RESEARCH.md "Email Template (Reservation Notification)" section
- Conditionally renders guest_email, guest_phone, message, totalPrice rows only if values are truthy

**Route 1: POST /:slug/reservations (PUBLIC — no auth)**
- No `requireUser` middleware — anonymous guests can submit
- Extract `slug` from params, resolve hotel by slug: `supabase.from('hotels').select('id, name, owner_email').eq('slug', slug).single()`
- If no hotel: return 404 `{ success: false, error: 'Hotel not found' }`
- Extract from req.body: `room_id`, `guest_name`, `guest_email`, `guest_phone`, `arrival_date`, `departure_date`, `nights`, `total_price`, `message`
- Validate: `guest_name` required (return 400 if missing), `arrival_date` required, `departure_date` required, `nights` must be > 0 (return 400 `'Išvykimo data turi būti vėlesnė nei atvykimo'` if not)
- Resolve room name for email: `supabase.from('hotel_rooms').select('name, number').eq('id', room_id).single()` — use `roomData?.name || 'Kambarys ' + roomData?.number` as roomName
- Insert: `supabase.from('hotel_reservations').insert({ hotel_id: hotel.id, room_id, guest_name, guest_email, guest_phone, arrival_date, departure_date, nights, total_price, message }).select().single()`
- If insert error: return 500
- Fire-and-forget email if `hotel.owner_email` exists:
  ```javascript
  if (hotel.owner_email) {
    emailService.sendReservationNotification({
      ownerEmail: hotel.owner_email, hotelName: hotel.name,
      roomName, guestName: guest_name, guestEmail: guest_email,
      guestPhone: guest_phone, arrivalDate: arrival_date,
      departureDate: departure_date, nights, totalPrice: total_price,
      message

  }
  ```
- Return 201: `{ success: true, reservation }`

**Route 2: GET /:slug/reservations (ADMIN — requireUser)**
- Uses `requireUser` middleware
- Call `verifyOwner(req, slug)` — handle not_found (404), forbidden (403)
- Query: `supabase.from('hotel_reservations').select('*, hotel_rooms(name, number)').eq('hotel_id', hotel.id).order('created_at', { ascending: false })`
- Return 200: `{ success: true, reservations: data || [] }`

**Route 3: PATCH /:slug/reservations/:reservationId/status (ADMIN — requireUser)**
- Uses `requireUser` middleware
- Extract `slug`, `reservationId` from params, `status` from req.body
- Call `verifyOwner(req, slug)` — handle not_found, forbidden
- Validate status: must be `'confirmed'` or `'cancelled'` — return 400 if invalid
- Update: `supabase.from('hotel_reservations').update({ status }).eq('id', reservationId).eq('hotel_id', hotel.id).select().single()`
- If no data returned (reservation not found or doesn't belong to hotel): return 404
- Return 200: `{ success: true, reservation: data }`

All routes wrapped in try/catch with `console.error` and 500 response on unexpected errors — same pattern as existing hotel routes.
